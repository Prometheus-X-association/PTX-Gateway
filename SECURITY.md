# Security Policy

This document describes how security issues should be reported and handled.

---

## Scope

This repository provides a reference implementation of a dataspace gateway.  
Security considerations include, but are not limited to:

- Authentication and authorization
- API exposure
- Dependency vulnerabilities
- Misconfiguration risks

---

## Reporting a Vulnerability

If you discover a security vulnerability:

- **Do not open a public issue**
- Report it privately to the project maintainers or designated consortium contact

Provide, where possible:

- Description of the issue
- Steps to reproduce
- Potential impact

---

## Handling and Disclosure

Security reports are reviewed on a best-effort basis.

- Fixes may be released without prior public disclosure
- No guaranteed response time is provided
- This repository does not operate a bug bounty programme

---

## Disclaimer

This software is provided as a research and innovation output.  
It is not guaranteed to meet production security requirements without further hardening.
