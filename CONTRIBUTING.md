# Contributing Guidelines

This repository is part of a **publicly funded EU project** and follows governance principles appropriate for collaborative, open innovation.

---

## Scope of Contributions

Contributions may include:

- Bug fixes and refactoring
- UI/UX improvements
- Security enhancements
- Documentation updates
- Dataspace-related extensions

All contributions are subject to review by project maintainers.

---

## Contribution Process

1. Fork the repository and create a feature branch.
2. Ensure your contribution:
   - Is clearly documented
   - Does not introduce security risks
   - Does not introduce incompatible licenses
3. Submit a pull request describing the changes and rationale.

---

## Review and Acceptance

All contributions are treated as **external contributions**, including those created using AI-assisted tools.

Reviews consider:

- Code quality and maintainability
- Security implications
- Architectural consistency with EDGE-Skills / PTX principles
- Licensing and dependency compliance

---

## AI-Assisted Contributions

AI-assisted development tools may be used.  
By submitting a contribution, you confirm that:

- You have reviewed the code
- The contribution does not knowingly include proprietary material
- Licensing obligations are respected

---

## Licensing of Contributions

By contributing, you agree that:

- Your contribution is licensed under the **MIT License**
- You have the right to submit it under these terms

No Contributor License Agreement (CLA) is required.

---

## Dependencies

New dependencies must:

- Use OSI-approved open-source licenses
- Be compatible with the MIT License
- Be documented in the pull request

---

## Governance

Final decisions regarding scope and acceptance rest with the EDGE-Skills project maintainers.
